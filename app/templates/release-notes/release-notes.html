Released: 5/8/2017

Introduction

In this release the DMC development team focused most efforts on revamping the many existing workflows around workspaces. As one may be able to guess from looking at our most recent releases, workspaces have received a layout face-lift. There is some new functionality when it come to applications that we think users will make running applications an even smoother experience than before.

Codebase

Vault Integration

  • We've integrated Vault into our deployment framework for our playform in this release. This change will not effect users directly but makes sure that our team is able to work on our platform virtual machines in a much safer capacity. Users as usual can rest assured that only those with the proper access level are working on updating the code base and infrastructure that supports the website.

Updated Terraform Infrastructure Deployment Scripts

  • Users that know our development team's infrastructure deployment methods know that we use Terraform to deploy and build our infrastructure. We've made some updates due to some recent changes that allows us to once more deploy our infrastructure in a more automated way.
  • Along with this update, we now have new code that allows us to now automatically build our ELK Stack/Stack Monitoring Infrastructure in tandem with the rest our our infrastructure.

Updated Terraform Version

  • We've updated our Terraform deployment scripts to now be compliant with Terraform Verson 0.9.4. If you are interested in deployiong our infrastructure yourself for this or other versions of the DMC going forward, you will need to make sure you have this version of Terraform installed.

File Signing (SHA256)

  • In order to give users the piece of mind that they are not downloading documents that have been altered after they have been uploaded to our site, we have implemented SHA256 for uploaded files. SHA256 is a cryptographic hash which is like a semi unique signature for a text or a data file. The signature is very difficult to regenerate exactly the same way once created, so our users can know that the files being downloaded are always the original and unaltered versions uploaded to the DMDII Projects' profile pages.

Released: 4/21/2017

Introduction

During this release, the DMC focused heavily on what we consider to be "Content Cleanup." In the last release users may have noticed that the DMDII Portal received a UI overhaul. In this release we focused on fixing a few issues that were brought to our attention by avid users of the Portal as well as other bugs we are aware of in the system.

Beyond bugs, we also worked on adding more general site features such as notifications, invitations, and update aggregation which we expect will help to drive our users to learn more about small updates happening with DMDII Members and DMDII Projects as well as being able to manage organization profiles and workspaces more effectively.

Codebase

  • Framework for More Secure Document Access

    • We are aware that some users of our platform want to make sure that the documents they are downloading/getting access to through workspaces and the DMDII Portal. In this release we worked to put together a framework that will be introduced in a upcoming release to make it harder for an individual to share information with someone that should not have access to these documents.

Applications

Marketplace Applications

We've include two new application in our Marketplace for our users to demo.This time the applications are coming from one of our DMDII Academic Members. North Carolina State University is working to build three more applications similar to these two. If you are a DMDII member, definitely check out their 15-16-08 project which can be found in the DMDII Project Directory.

    Learn more about the apps when visiting the DMC Marketplace or at their associated Confluence pages:
    • Machine Status: This app is designed to retrieve status of machines in the NCSU ISE Manufacturing Laboratory.
      • App documentation can be found at: Machine Status by North Carolina State University
    • Machining Data: This app retrieves detailed machining dataset associated with any of the 3 machines in current dataset.
      • App documentation can be found at: Machining Data by North Carolina State University
    • Machining Data with Plots: This app retrieves a machining dataset and plots values as an XY graph.
      • App documentation can be found at: Machining Data with Plots by North Carolina State University

Updates to Existing Applications

We've been told in the past that many of the applications we've published to our marketplace during our beta release are very ambiguous and that most people do not know what these applications do. In a continuous effort, we have begun to update application names and descriptions to be more descriptive to help users more decisively use the marketplace applications available to them on the DMC.

  • Changes made thus far include:
    • Our Beta Factor application has been renamed Nonlinear Crack Growth Rate Correction Factor Calculation, and its documentation has also been updated.
    • Our K Max application has been renamed Critical Stress Intensity Factor, and its documentation has also been updated.

Installing and Launching DOME

We have found recently that our instructions for users being able to install and launch our DOME SDK on Windows and Mac operating systems were no longer working. We've made a few updates to our documentation to account for this. Individuals interested in building DOME applications may find it useful to revisist our update documentation found on our Confluence wiki page: Installing and Launching DOME

DOME and External APIs

We understand that many developers that will be building DOME applications may need to take advantage of calling external APIs in order to make their applications function properly or in the architecture they've designed. In order to easy the process of doing this. We've created documentation that users should be able to reference to allow them to do exactly that. The instructions to do this can be found on our Confluence Wiki page: DOME Applications and External APIs

Released: 4/7/2017

Introduction

During this release, the DMC heaviliy focused on making updates that effected the DMDII Membership Portal. The Membership Portal recieved a new look and feel (navigation hasn't changed too much so don't worry if you were a fan of how you traversed the portal before). In addition to these changes, we are slowly making the shift to releasing new content with every release we put out. We've also made a few updates to our legal page so definitely look over this when you have the opportunity.

As with most releases, we have added three more application to the DMC Marketplace in the form of external application contributions from DMDII Member Organizations such as Integris Engineering and Pennsylvania State University. Read more about them in our Applications section found below.

Finally, if you couldn't tell already, all release notes about the platform and the website will be published to this page going forward, so you no longer need to leave the DMC to learn more about the hard work that is put into each release as well as what new features and content is available to you every two weeks. Codebase

White-list and Black-List Framework

  • The DMC has created a framework to be able to white-list and black-list endpoints. In the past these endpoints were listed one by one, but now we will be able to white-list and black-list endpoints via a singular file. In future updates we will be utilizing this functionality to add full lists of white and blacklist on our own servers.

Quarantine Bucket Workflow Framework

  • We have made changes to our framework for our document upload process to make it even safer. In the past, our infrastructure would scan all documents being uploaded to our site before making the document available for user to download. Documents that did not pass the virus and malware scan process would be marked as not safe and would not become available for download through our site. When implimented, this will move files marked as not safe by our infrastructure will be moved into our quarentine storage solution. We will at this point be able to analyze in depth the threats our site experiences on a regular bases, to ensure our users are always safe when using our platform. This functionality will added in a future update while we are going through the process of testing this fuctionality as best we can befor adding it to our production platform.

DMC License files have been updated

  • The DMC has updated all repositories stored on GitHub with new License Files. These Files can be found in the root of all repositories. The GitHub link can be found at: https://github.com/dmc-uilabs

  • Conceptual Cost Model (Pennsylvania State University): Estimates the cost of a part or assembly.

    • App documentation can be found at: Weld Cost Model by Pennsylvania State University

Released: 1/20/2017

With the success of the Alpha release just a little bit over a month ago we are introducing our Beta release of the Digital Manufacturing Commons platform.
